# Container Publishing Pipeline

Tyrum images are now published by the `publish-containers` GitHub Actions
workflow. It covers both nightly pre-release images for fast testing and
tagged production releases.

## Nightly pre-release builds
- Runs every day at 03:15 UTC on the current `main` head.
- Uses per-image path filters to decide whether a container changed in the last
  24 hours; unchanged services are skipped to avoid wasted rebuilds.
- Publishes `ghcr.io/rhernaus/tyrum-<service>:nightly`,
  `:sha-<12-char-commit>`, and a monotonic `:build-<run-id>` tag for
  reproducibility.
- Builds share BuildKit cache layers via `ghcr.io/rhernaus/cache/<service>:buildcache`
  so only the layers touched by recent commits are rebuilt.
- Each pushed image is scanned with Trivy and an SPDX SBOM (generated by Syft)
  is uploaded as a short-lived artifact (`sbom-<service>-<run-id>.spdx.json`).

For manual nightly publishes (e.g. after backfilling a branch), use the
workflow’s `workflow_dispatch` entry and leave the channel set to `nightly`.

## Production releases
- Triggered by pushing a semver tag (`vMAJOR.MINOR.PATCH`) that points at the
  vetted commit.
- Produces immutable tags `:<tag>`, `:<major>.<minor>`, `:<major>`, plus a
  `:sha-<12-char-commit>` convenience tag. The first release push for a major
  stream creates the `<major>` tag.
- Runs the same BuildKit, SBOM, and vulnerability scanning steps as the nightly
  job, so promotion uses the exact artifact set that passed those checks.

To rerun a production publish without creating a new git tag, dispatch the
workflow manually, choose the `release` channel, and supply the existing tag in
the `release_tag` field. The workflow will re-push the canonical tags for that
commit.

## Operational notes
- GHCR authentication uses the Actions-provided `GITHUB_TOKEN`; ensure the
  repository retains `packages: write` permission for workflows.
- Nightly runs skip unchanged services; if a container must be rebuilt despite
  no recent code changes (e.g. base image CVE), dispatch the workflow manually
  for that day.
- SBOM artifacts persist for five days. Download them from the workflow run
  when submitting audits or provenance evidence.
- Trivy currently fails the job on HIGH or CRITICAL findings. If a false
  positive blocks a release, suppress it with a temporary ignore rule and
  document the rationale in the release notes.
